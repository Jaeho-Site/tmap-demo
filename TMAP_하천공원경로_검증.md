# TMAP 보행자 경로 — 하천변/공원 산책로 포함 여부 검증

> 작성일: 2026-07-06 · 방법: TMAP 보행자 경로안내 API 실호출(대전 10개 지점쌍)
> 목적: 네이버/카카오는 하천 둔치·공원 산책로로 경로를 추천하는데, TMAP도 그런지 실증

---

## 결론 (한 줄)

**TMAP 보행자 경로는 하천 둔치 산책로·공원 내부/산 산책로를 경로 네트워크로 보유하지 않는다.**
그런 지점을 넣으면 **가장 가까운 "도로"로 끌어붙여(스냅)** 도로 위로만 경로를 만든다. 사용자 관찰이 맞다.

---

## 판정 기준

- **스냅 거리**: 입력 좌표 → TMAP이 실제로 잡은 시작/종료점 거리. 크면 그 자리에 보행 네트워크 없음.
- **우회 배율**: 경로거리 ÷ 직선거리.
- **경로 안내문 / facilityType**: 실제로 어떤 길을 탔는지. `12`=독립 보행자도로, `11`=도로변 보도, `1`=교량, `15`=육교.

## 결정적 근거 — 실제 안내문

**갑천 둔치** (둔치 한가운데 지점 → 202m 밖 도로로 스냅):
```
· 보행자도로를 따라 279m 이동
· 우회전 후 대덕대로를 따라 564m 이동   ← 하천이 아닌 왕복 대로로 탈출
· 좌회전 후 404m 이동 …
```

**보문산 산책로** (626m 직선을 4.7km로):
```
· 보행자도로를 따라 1964m 이동
· 좌회전 후 사정공원로를 따라 407m …
· 좌회전 후 보문산공원로를 따라 1008m 이동   ← 산을 빙 둘러 도로로
```

**유등천 둔치**: 안내문 전부 `태평로 / 동서대로 / 유등로` + 횡단보도 → **둔치 산책로는 한 번도 안 씀.**

**공통**: 8개 경로의 구간 facilityType은 전부 `11`(도로변 보도)·`1`(교량)·`15`(육교).
**독립 보행자도로 코드 `12`, 하천/공원 산책로는 단 한 구간도 나오지 않음.**

---

## 시사점 (프로젝트)

1. **TMAP만으로는 하천변·공원 산책로 추천 불가.** 네이버/카카오가 강한 지점이 바로 이 부분.
2. 따라서 **공원·가로수길·보행자전용도로를 자체 노드/경로로 쌓는 현재 방향이 옳다.**
3. 보완 필요 데이터: **하천 둔치 산책로 라인(갑천·유등천·대전천 자전거/보행 노선), 공원 내부 보행로, 등산로(OSM `highway=path/footway`)**.
   - TMAP은 이 구간들의 "도로 접속부"까지만 안내에 쓸 수 있으므로, 트레일 본선은 OSM/지자체 GPX로 직접 확보해 우리 그래프에 넣어야 한다.
4. TMAP 보행 경로는 **도로변 이동(노드 간 접근)** 용도로 계속 유효 — 트레일 데이터와 하이브리드로 쓰는 게 맞다.
