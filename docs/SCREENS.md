# 산책온 — 화면 명세 (MVP)

> **문서 상태:** v1.0 · **작성일:** 2026-07-10
> **함께 볼 문서:** [PRD.md](../PRD.md) §5.3, [USER_STORIES.md](../USER_STORIES.md), [DESIGN_SYSTEM.md](./DESIGN_SYSTEM.md)
> **읽기:** 각 화면 = 목적 · 핵심 컴포넌트 · 주 CTA · 상태 · 연결 스토리. 우선순위는 PRD를 따른다(P0/P1).

**주 내비:** 하단 5탭 **홈 · 탐험 · 기록 · 저장 · 프로필**. 산책 루프(06→07→08→10→11)는 홈에서 여는 **스택 플로우**.

---

## 온보딩 / 진입

### 01 스플래시 (P0)
- 목적: 브랜드 첫인상. 워드마크(라임그린·라운드 폰트) + 태그라인 "지금 있는 그 자리에서, 바로 걷기 시작하세요."
- 상태: 자동 전환(1s) → 위치 권한 또는 홈(게스트).

### 02 위치 권한 / 온보딩 (P0) · US-1.1
- 목적: OS 다이얼로그 **이전에** *왜* 위치가 필요한지 쉬운 말 안내.
- 컴포넌트: `r5/r6` 장식 이미지, 설명, `Button.primary`("현위치로 시작"), 텍스트 링크("위치 없이 둘러보기").
- 상태: 허용→위치+정확도 확보 후 홈 / 거부→수동 위치 핀 폴백(막다른 길 없음).

### 03 로그인 / 회원가입 (건너뛰기 가능) (P1) · US-1.5
- 목적: 저마찰 계정(기록 누적). **절대 "산책 시작"을 가입 뒤로 막지 않음**(UX 가드레일 1).
- 컴포넌트: 소셜/이메일(목업), 큰 "게스트로 시작".

## 핵심 루프

### 04 홈 (P0) · US-1.2, 2.1
- 목적: 3초 안에 **대표 추천 코스 1개**. 선택 마비 방지(한 번에 하나).
- 컴포넌트: 인사/날씨 넛지(선택), **대표 추천 `Card.course`**(썸네일·코스명·거리·시간·난이도·이유 한 줄), `Button.primary`("산책 시작"), 보조("다른 코스 보기"), 조건 진입 버튼.
- 상태: 로딩(≤3s, 진행 표시) / 결과 / 위치 없음(수동 핀 유도).

### 05 조건 시트 (P0) · US-1.3, 1.4
- 목적: 선택적 조건. 무입력도 OK.
- 컴포넌트: `Sheet` + 시간 프리셋 칩(10/20/30) · 거리 `Slider` · 난이도 · **목적 칩**(🌳🛡️🤫🦵 다중선택). 얇은 목적엔 신뢰도 힌트.
- CTA: "이 조건으로 추천".

### 06 추천 (P0) · US-2.1~2.4
- 목적: 대표 추천 + 근거 있는 이유 + 탈출구.
- 컴포넌트: 코스 카드(지도 미리보기), **자연어 이유**(실제 속성), "다른 코스 보기"(재추천=암묵 신호).
- 상태: 대안 로딩 / 후보 소진("조건을 넓혀보세요").

### 07 추천 상세 (P0) · US-2.3, 7.1
- 목적: 결정 전 전체 파악.
- 컴포넌트: **지도**(출발→경유→반환→도착, TMAP 폴리라인), 거리·예상시간, 목적 적합도, **신뢰도/한계 배지**, 핵심 지점 리스트, `Button.primary`("산책 시작"), ♡ 저장.

### 08 산책 중 (진행) (P0) · US-3.1~3.3
- 목적: 저주의 핵심 지점 안내. 몸/주의 존중.
- 컴포넌트: 지도(현위치·다음 지점), 상단 **타이머/거리**, **항상 보이는 "산책 종료"**, 비모달 메모 입력, (P1) "여기 문제" 핀 버튼. dev: GPS 시뮬레이터 토글.
- 상태: 진행 / 이탈 자동 질문("길 막힘/없음?", 닫기 가능) / 완주 감지 / 수동 종료.

### 09 문제 신고(핀) 시트 (P1) · US-3.4
- 목적: 현 위치에 원탭 핀 + 문제 유형.
- 컴포넌트: `Sheet`, 유형 칩(🚧공사·🚫길없음·기타), 확인. → 가장 가까운 구간 신뢰도 감소.

### 10 산책 완료 / 피드백 (P0) · US-4.1, 4.2
- 목적: 기억 생생할 때 원탭.
- 컴포넌트: 요약(거리·시간), **👍/👎 대형 버튼**, 👎(또는 상시) **태그 시트**(🚧🔇🚫🌑⚠️ + 직접입력), 전부 건너뛰기 가능.

### 11 AI 회고 / 산책 요약 (P0) · US-5.1
- 목적: 기록이 잡일이 아니라 보상.
- 컴포넌트: **따뜻한 회고 텍스트**(목업/템플릿, 실제 데이터 근거), 경로 지도, 통계(거리·시간·칼로리), 저장/공유. "당신 피드백이 X 했어요"(있으면). → 홈.

## 리텐션 & 계정

### 12 기록(히스토리) (P0) · US-6.1, 6.2
- 목적: 투자 가시화.
- 컴포넌트: 상단 `Tabs`(나/연속기록), 산책 리스트 카드, **연속기록 배너**, **월간 리포트**. 빈 상태는 `r*` 장식 + "첫 산책을 시작해요".

### 13 대전 탐험 (도감 / 지역 지도) (P1) · US-6.3
- 목적: 발견 유도 → 피드백 수집.
- 컴포넌트: 구/동 **탐험 % 지도**, 배지 그리드, 경로 도감(희귀도), 미검증 경로 강조.

### 14 저장 (P1) · US-6.4
- 목적: 좋아한 코스 재방문.
- 컴포넌트: AllTrails Saved 스타일 — Lists/Custom routes/Favorites 그리드 카드, 정렬, ＋추가.

### 15 프로필 / 설정 (P0)
- 목적: 계정·환경설정.
- 컴포넌트: 프로필 헤더, 로그북 진입(기록 요약), 설정(위치 최소수집·알림·개인정보), 게스트→로그인 유도.

## 어드민 (P0, 별도 트랙 — 프로토타입 후)

- **A1 코스 관리:** 시딩 코스 조회/큐레이션.
- **A2 AI 검토:** 추출 태그·위험·구간 신뢰도 점검.
- **A3 통계:** 퍼널(시작·선택·완주·피드백률·재방문) + 데이터 건강도·커버리지.

## 화면 플로우 (요약)

```
스플래시 → 위치권한 → (로그인/건너뛰기=게스트) → 홈
   홈 ─(조건)→ 조건시트 ─→ 추천 ─("다른 코스")↺
                              │
                              ▼ 추천상세 ─(산책 시작)→ 산책중
   문제신고시트 ◄("여기 문제")── 산책중 ──("길 막힘?" 자동질문)
                              │ (완주/수동종료)
                              ▼ 완료·피드백(👍/👎→태그) → AI 회고 → 홈
리텐션 진입: 좋은날 푸시→홈 · 기록▸연속기록 · 탐험▸미검증 경로→추천상세
```
