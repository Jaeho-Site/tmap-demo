- **모바일 우선 480px 프레임:** 모든 화면은 최대 480px 폰 프레임(넓은 화면에선 검은 레터박스 위 중앙 정렬). 하단 탭 없는 풀스크린 스택(코스/산책/완료)도 `PhoneFrame`으로 동일 폭 유지 → **어떤 페이지도 모바일 레이아웃을 벗어나지 않음.**
  - `components/layout/AppShell.tsx`, `components/layout/PhoneFrame.tsx`.
- **하단 5탭:** 홈 · 탐험 · 기록 · 저장 · 프로필. `BottomTabBar.tsx`.

  - `index.css`(디자인 토큰), `components/ui/*`.

## 1. 열 가지 원칙 — 구현 근거 (우선순위 순)

### P1 — 수 초 안에 걷기 시작 ✅
- **홈 상단 "빠른 시작" 박스**: 단 **1탭**으로 MVP 핵심 진입 — `AI 경로 추천`(모험 폼 자동 오픈) / `자유 산책`.
- 조건 미설정이어도 위치 기반 기본 추천 즉시 표시. 게스트도 전부 사용 가능.
- 근거: `pages/HomePage.tsx`, `pages/explore/AdventureView.tsx`.

### P2 — 한 번에 하나의 결정 ✅
- 홈은 **단일 대표 추천 카드** + "다른 코스" 단일 탈출구(긴 목록·필터·비교표 없음).
- 코스 상세는 점진적 공개(지도/이유 먼저, 핵심 지점·통계는 스크롤).
- 근거: `HomePage.tsx`, `CourseDetailPage.tsx`.

### P3 — 항상 이유를 설명하라 ✅
- 모든 추천에 **실제 코스 속성 기반 이유**(지역·거리·시간·난이도·목적 신뢰도) + 선택 조건 반영.
- **규칙 기반 랭킹(LLM 아님)** — 환각 방지 가드레일. 지도 API가 소유한 경로/거리만 인용.
- 근거: `features/recommend/reason.ts`, `features/recommend/engine.ts`.

### P4 — 피드백은 탭이지 에세이가 아니다 ✅
- 마찰 사다리 1→2단계만으로 동작: **원탭 👍/👎 → 아이콘 태그 시트**. 모든 단계 건너뛰기 허용, 완료를 텍스트로 막지 않음.
- 근거: `pages/WalkCompletePage.tsx`, `features/feedback/tags.ts`.

### P5 — 적절한 순간에 물어라 ✅
- **완주 즉시** 평가 화면 노출. 산책 중 명확한 이탈 시 **닫기 가능한 자동 질문**("길 막힘/없음?") — 비차단.
- 근거: `WalkCompletePage.tsx`, `WalkPage.tsx`, `useWalkSession.ts`.

### P6 — 기여가 기분 좋게 ◐
- **이타심(구현):** 문제 태그가 코스 품질에 누적 → 다음 걷는 이의 코스 상세에 주의 배너로 반영(순환이 눈에 보임).
- **상호성(부분):** 개인 피드백을 내 다음 추천에 반영하는 학습은 미구현.
- 근거: `features/feedback/quality.ts` → `CourseDetailPage.tsx`.

### P7 — 신뢰도에 정직하라 ✅
- 목적별 **신뢰도 배지**(검증/추정/낮음), 얇은 데이터 목적(조용함·관절)엔 이유에 유의 문구.
- AI 경로는 "베타 · 예시 경로(실 지형 미반영)"임을 명시.
- 근거: `components/ui/Badge.tsx`, `reason.ts`, `features/adventure/suggestRoute.ts`.

### P8 — 중독이 아니라 누적을 보상 ✅
- 자동 산책 기록, 연속·월간 통계, 코스/커스텀 경로 저장, 탐험 도감·배지 — **투자 누적**으로 리텐션. 다크 패턴·긴박감 없음.
- 근거: `features/history/*`, `store/saved.ts`, `store/customRoutes.ts`, `features/explore/collection.ts`.

### P9 — 걷는 이의 주의와 몸을 존중 ✅
- 턴바이턴 대신 **핵심 지점 안내**, 큰 탭 타깃, 한눈에 보는 시간·거리, **비모달 한 줄 메모**, **항상 가능한 수동 종료**.
- **GPS 오차 고려:** 허용 반경 35m + 흔들림 필터 2m + 수동 오버라이드로 거짓 "중단" 방지.
- 근거: `WalkPage.tsx`, `useWalkSession.ts`.

### P10 — 기본이 신뢰할 수 있게 ◐
- **최소 위치 수집**(활성 산책 중에만 watch, 그 외 지속 추적 없음). 안전(방범) 목적을 추천에 반영.
- 정직한 알림(좋은 날 푸시)은 **미구현** — 홈 날씨는 목업 표시.
- 근거: `hooks/useGeolocation.ts`, `useWalkSession.ts`.

## 3. 콘텐츠 & 보이스 (구현 준수)
- **톤:** 따뜻·격려·간결(친근한 동네 사람). 예: "지금, 여기서 · 바로 걷기 시작해요", "첫 산책을 시작해볼까요?".
- **근거 있게, 과장 없이:** 이유·회고는 실제 코스/기록 데이터만 인용.
- **한계에 정직하게:** "AI 추천 · 베타", "예시 경로", 얇은 데이터 유의 문구.
- **임팩트/상태를 숫자로:** 걸음수·칼로리·거리·연속일·정복도 %.

---

## 4. 접근성 & 환경 (구현 수준)
- **모바일 우선·큰 탭 타깃·고대비 다크 테마·라운드 폰트.** 색상+아이콘/라벨 병행(목적·신뢰도).
- **중단 가능:** 산책 중 메모/피드백이 산책을 막지 않음, 수동 종료 상시.
- **환경 관대:** 위치 거부 시 폴백, GPS 오차 허용 반경·시뮬레이터.
- (미검증: 실제 야외 햇빛 가독성/실기기 GPS 정확도.)

## 5. 디자인 가드레일 — 준수 현황

| 가드레일 | 상태 |
|---|---|
| 계정 생성 뒤로 "산책 시작"을 막지 않는다 | ✅ 게스트·목업 로그인 즉시 진입 |
| 근거 없는 추천을 보여주지 않는다 | ✅ 이유 항상 표시 |
| 피드백 완료에 타이핑을 요구하지 않는다 | ✅ 원탭+태그 |
| 데이터가 뒷받침 못 하는 속성을 주장하지 않는다 | ✅ 규칙 기반·신뢰도 표시 |
| 항상 수동 "산책 종료"를 제공한다 | ✅ |
| 중단/완주 기록 전 GPS 오차를 고려한다 | ✅ 허용 반경+필터+수동 |
| 좋은 날이 아니면 푸시하지 않는다 | ✗ 푸시 미구현(해당 없음) |
| 저밀도 지역에 소셜을 출시하지 않는다 | ✅ 소셜 기능 미출시 |

## 6. 남은 UX 과제 (이후)
- 개인 피드백 학습 → 추천 반영(상호성 완성, P6).
- 산책 중 "여기 문제" 핀 신고(P5).
- 좋은 날 넛지 푸시(P10) — 실제 날씨/대기질 연동.
- 코스 명명/편집(P8), 기여 임팩트 가시화(P6).
- 실 LLM 회고·보행자 라우팅 연동(정직성 유지하며 목업 대체).
