# 산책온 (SanchaekOn) — 구현된 MVP 사용자 스토리 (as-built)

> **문서 상태:** 구현 반영 v1.0 · **기준:** 현재 코드베이스에 실제로 동작하는 기능
> **함께 볼 문서:** [PRD.md](../PRD.md), [USER_STORIES.md](../USER_STORIES.md)(초안·전체 로드맵), [MVP_UX.md](./MVP_UX.md)
>
> 루트의 [USER_STORIES.md](../USER_STORIES.md)는 제품 로드맵 초안이다. 이 문서는 그중 **프론트엔드 프로토타입으로 실제 구현된 것**만 추려, 구현 위치·동작·한계를 기록한다.
> 범위: **프론트엔드 우선 프로토타입** — 규칙 기반 추천, 목업 AI, 실제 대전 코스 데이터, `localStorage` 영속화. 백엔드/실인증/서버 라우팅 없음.

## 2. 에픽별 사용자 스토리 (구현 기준)

### E0 — 앱 시작 & 로그인

**US-0.1 ✅ 실제 앱 같은 시작 경험**
*사용자로서 나는 앱을 열면 깔끔한 시작 화면을 보고 싶다.*
- 스플래시: 로고 팝인 + 펄스 링 + 워드마크/태그라인 등장(약 2초).
- 구현: `components/startup/SplashScreen.tsx`, 키프레임 `index.css`.

**US-0.2 ✅ 소셜 로그인으로 진입 (목업)**
*사용자로서 나는 카카오/네이버/구글로 간편히 시작하고 싶다.*
- 카카오·네이버·구글 브랜드 버튼 + "로그인 없이 둘러보기"(게스트).
- **어떤 버튼이든 실제 인증 없이 바로 진입**하고 provider를 `localStorage(sanchaek.auth)`에 저장.
- 재실행 시: 스플래시 → (인증돼 있으면) 로그인 생략하고 앱으로 **리다이렉트**. 미인증이면 로그인 화면.
- 로그아웃: 프로필 → 로그인 화면 재진입.
- 구현: `app/AppRoot.tsx`, `components/startup/LoginScreen.tsx`, `pages/ProfilePage.tsx`.

### E1 — 온보딩, 위치 & 조건

**US-1.1 ✅ 위치 한 번 허용 → 즉시 위치 기반 추천**
- 홈 진입 시 `getCurrentPosition` 1회 요청. 허용 시 현위치를, 거부 시 대전 중심(`DAEJEON_CENTER`)으로 우아하게 폴백.
- 지속 백그라운드 추적 없음(활성 산책 중에만 watch).
- 구현: `hooks/useGeolocation.ts`, `pages/HomePage.tsx`.

**US-1.2 ✅ 거의 입력 없는 기본 추천**
- 조건 미설정이어도 위치 기반 기본 추천 1개 즉시 제공. 조건은 선택 사항.
- 구현: `features/recommend/engine.ts`, `HomePage.tsx`.

**US-1.3 ✅ 산책 목적 선택**
- 목적 태그: 🛡️ 방범·안전 / 🌳 자연친화 / 🦵 관절보호 / 🤫 조용함 (다중 선택).
- 홈 상단 칩 또는 조건 시트에서 선택.
- 구현: `data/daejeonData.ts`(PURPOSES), `store/conditions.ts`.

**US-1.4 ✅ 시간 / 난이도 조건**
- 조건 시트: 목표 시간, 난이도(easy/moderate/hard), 목적. 랭킹에 반영.
- 구현: `components/recommend/ConditionSheet.tsx`, `store/conditions.ts`.

**US-1.5 ◐ 경량 계정 & 기록 유지**
- 소셜 로그인/게스트 진입(목업). 산책·저장·연속 기록은 계정과 무관하게 `localStorage`에 누적(실계정 연동은 미구현).

### E2 — 추천 & 설명

**US-2.1 ✅ 강력한 추천 하나, 빠르게**
- 홈은 **단일 대표 추천 카드** 먼저 표시(긴 목록 아님).
- 구현: `HomePage.tsx`(오늘의 추천 코스).

**US-2.2 ✅ "왜 이 코스인지" 근거 있는 이유**
- 이유는 **실제 코스 속성**(지역·거리·시간·난이도·목적 신뢰도)과 사용자 조건을 조합한 템플릿. 지어낸 속성 없음.
- 데이터가 얇은 목적(조용함·관절)에는 정직한 유의 문구를 덧붙임.
- 구현: `features/recommend/reason.ts`.

**US-2.3 ✅ 결정 전 코스 상세 보기**
- 코스 상세: 지도, 핵심 지점(출발/경유/반환/도착), 거리·예상 시간, 목적별 **신뢰도 배지**(검증/추정/낮음), 걷는 이들이 남긴 **주의 사항**, 하트 저장.
- 구현: `pages/CourseDetailPage.tsx`.

**US-2.4 ✅ 거부하고 재추천**
- "다른 코스" 한 번 탭으로 다음 후보. 재추천은 암묵 `skip` 신호로 기록.
- 구현: `HomePage.tsx`, `features/feedback/signals.ts`.

**US-2.5 ✗ 과거 피드백 반영 언급** — 미구현(랭킹에 개인 피드백 학습 미반영).

**랭킹 로직(구현):** 근접성(가중치 2, 5km 이내) + 목적 일치(개당 3, 불일치 −1.5) + 시간 적합(3) + 난이도 일치(±) + 검증 신뢰도 보너스(0.5). **LLM 아님 — 환각 방지.** `features/recommend/engine.ts`.

### E3 — 산책 진행

**US-3.1 ✅ 핵심 지점 안내 (턴바이턴 과부하 아님)**
- 진행: 출발 → 경유 → 반환 → 도착. 다음 핵심 지점 라벨을 상단에 표시.
- 구현: `pages/WalkPage.tsx`, `features/walk/useWalkSession.ts`.

**US-3.2 ✅ GPS 오차 대비 완주 감지 + 항상 가능한 수동 종료**
- 완주/이탈: 핵심 지점 허용 반경 35m, 이탈 임계 45m, GPS 흔들림 필터 2m.
- **"산책 종료"** 수동 컨트롤 상시 노출(폴백). 실기기 없이 데모용 **시뮬레이터** 제공.
- 구현: `useWalkSession.ts`, `WalkPage.tsx`.

**US-3.3 ✅ 산책 중 한 줄 메모**
- 비모달 한 줄 메모 입력, 칩으로 누적. 산책을 막지 않음.
- 구현: `WalkPage.tsx`.

**US-3.5 ◐ 경로 이탈 자동 질문**
- 지속 이탈 시 "여기 길이 막혔거나 없나요?" 닫기 가능한 프롬프트 노출.
- (닫기만 구현. "예" 확정→구간 신고 연결은 미구현.)

**US-3.4 ✗ "여기 문제" 핀 신고** — 미구현(P1 이월).

### E3′ — 모험: 사용자가 만드는 경로 (탐험 탭)

**US-3M.1 ✅ 자유 산책 (프리폼 경로 기록)**
- 정해진 코스 없이 시작 → 실시간 GPS(또는 시뮬)로 걸은 경로를 그려가며 거리·시간 기록 → 종료.
- 구현: `pages/explore/AdventureView.tsx`, `features/walk/useFreeWalk.ts`.

**US-3M.2 ✅ AI 경로 추천 (조건 폼)**
- 폼: **거리(1/2/3/5km) · 왕복/편도 · 테마(멍때리기/자연/도심/운동/조용한 길) · 자유 메모**.
- 조건에 맞춰 현위치 기준 순환/편도 경로 생성 → 미리보기 → 시작.
- **정직성:** "AI 추천 · 베타" 표기 + 실제 도로·지형 미반영 예시 경로임을 요약에 명시. 생성기는 기하 알고리즘(목업), 실 LLM/보행자 라우팅으로 교체 가능한 구조.
- 구현: `features/adventure/suggestRoute.ts`, `AdventureView.tsx`.

**US-3M.3 ✅ 커스텀 경로 저장 & 재산책**
- AI 미리보기 하트 저장, 또는 산책 완료 후 걸은 트랙 저장.
- 저장 탭 "내가 만든 경로" → 탭하면 경로를 다시 불러와 재산책.
- 구현: `store/customRoutes.ts`, `AdventureView.tsx`, `WalkCompletePage.tsx`, `SavedPage.tsx`.

### E4 — 피드백 포착

**US-4.1 ✅ 완주 직후 원탭 평가**
- 완료 화면에서 즉시 👍 / 👎. 한 번의 탭으로 최소 피드백 완료.
- 구현: `pages/WalkCompletePage.tsx`.

**US-4.2 ✅ 타이핑 없는 태그**
- 👎: `🚧 공사중 · 🔇 시끄러움 · 🚫 길없음 · 🌑 어두움 · ⚠️ 위험` + 직접 입력.
- 👍: `🌳 경치 좋음 · 🛡️ 안전함 · 🤫 조용함 · 🪑 쉼터 많음 · 😀 걷기 편함`.
- 태그는 선택 사항, 건너뛰어도 평가 기록.
- 구현: `features/feedback/tags.ts`, `WalkCompletePage.tsx`.

**US-4.4 ✅ 암묵 신호 학습(포착)**
- `view / skip / start / complete / abandon` 신호를 기록(향후 개선용).
- 구현: `features/feedback/signals.ts`.

**US-4.3 ✗ 사진/자유 텍스트 첨부** — 미구현(한 줄 텍스트만).

### E5 — AI 회고 & 코스 품질

**US-5.1 ◐ 개인 AI 회고 (목업)**
- 완료 후 산책 기록(거리·시간·완주 여부) + 첫 메모를 인용해 따뜻한 요약 생성.
- **규칙 기반 목업**(실제 LLM 아님) — 실제 일어난 데이터에만 근거해 지어낸 사건 없음.
- 구현: `features/retrospective/generate.ts`.

**US-5.2 ◐ 피드백 → 구조화 태그**
- 사전 정의 태그 분류를 그대로 사용(자유 텍스트 추출은 미구현).

**US-5.3 ✅ 주의 사항을 코스 정보로**
- 문제 태그가 코스 품질에 누적되어 **코스 상세의 주의 배너**로 노출.
- 구현: `features/feedback/quality.ts` → `CourseDetailPage.tsx`.

### E6 — 리텐션

**US-6.1 ✅ 자동 산책 기록**
- 완주/종료 시 경로·거리·시간·(추정)칼로리 자동 기록.
- 구현: `features/history/records.ts`, `WalkPage.tsx`.

**US-6.2 ✅ 통계 · 연속 기록**
- 총/이번 달/**오늘** 산책·거리, 연속일. 홈 "오늘의 활동"에 걸음수·칼로리·거리 + 연속.

**US-6.3 ✅ 대전 탐험 도감**
- 전체 정복도 %, 구별(유성/서구/중구/동구/대덕) 진행률, 경로 도감(희귀도 일반/희귀/영웅/전설 — 거리 기준).
- 배지 6종: 🥾 첫 걸음 · 🏅 완주왕 · 🌅 새벽 산책러 · ✍️ 기록가 · 🤝 피드백 파트너 · 🗺️ 대전 탐험가.

**US-6.4 ✅ 코스 저장 (+ 커스텀 경로)**
- 카탈로그 코스 하트 저장, 커스텀 경로 저장. 저장 탭에 모임.
- 명명/편집은 미구현(저장·재산책까지).
- 구현: `store/saved.ts`, `store/customRoutes.ts`, `SavedPage.tsx`.

**US-6.5 ✗ 좋은 날 넛지(푸시)** — 미구현. (홈 날씨 위젯은 **목업 표시**만.)

### E7 — 신뢰 & 신뢰도

**US-7.1 ✅ 과잉 약속이 아닌 정직한 신뢰도**
- 코스 상세에 목적별 신뢰도 배지(검증/추정/낮음). 얇은 데이터 목적은 이유에 유의 문구.
- 구현: `CourseDetailPage.tsx`, `reason.ts`, `components/ui/Badge.tsx`.

**US-7.2 ✅ 교차 검증(1건=의심 / 3건=확정)**
- 문제 태그가 코스별로 누적되어 1건=의심, 3건+=확정 등급으로 주의 배너에 표시.
- 구현: `features/feedback/quality.ts`.

**US-7.3~7.5 ✗** 신고 만료 · 기여 임팩트 · 어뷰징 필터 — 미구현.

## 3. 엔드투엔드 플로우 (as-built)

```
스플래시 ──► 소셜 로그인(또는 게스트) ──► 홈
   │                                        │
   │              ┌─────────────────────────┼──────────────────────────┐
   │              ▼                          ▼                          ▼
   │        빠른 시작(1탭)             오늘의 추천 코스            탐험 ▸ 모험
   │        · AI 경로 추천 폼          · 이유 + 신뢰도            · 자유 산책
   │        · 자유 산책               · 다른 코스                · AI 경로 추천
   │              │                          │                          │
   │              └───────────► 코스/경로 상세 ◄─────────────────────────┘
   │                                    │  (산책 시작)
   │                                    ▼
   │            (이탈 자동질문) ◄──── 산책 진행 (핵심지점·GPS/시뮬·수동 종료·메모)
   │                                    │  (완주 / 종료)
   │                                    ▼
   │               산책 완료 ── 원탭 👍/👎 → 태그 → AI 회고(목업) → 경로 저장
   │                                    │
   └──── 리텐션: 홈 오늘의 활동 · 기록/연속 · 탐험 도감 · 저장(내 경로) · 프로필
```

**핵심 루프:** 추천/생성 → 산책 → 피드백 → 회고/품질개선 → 재방문.

## 4. 데이터 저장 (localStorage)

| 키 | 내용 |
|---|---|
| `sanchaek.auth` | 로그인 provider(kakao/naver/google/guest) |
| `sanchaek.walks` | 산책 기록(경로·거리·시간·평가·태그·회고) |
| `sanchaek.signals` | 암묵 신호(view/skip/start/complete/abandon) |
| `sanchaek.coursequality` | 코스별 문제 태그 카운트(의심/확정) |
| `sanchaek.saved` | 저장한 카탈로그 코스 id |
| `sanchaek.customRoutes` | 저장한 커스텀 경로(자유/AI, 경로 데이터 전체) |

## 5. 알려진 한계 (정직성)

- **AI는 목업:** 회고·경로 추천은 규칙/기하 알고리즘. 실 LLM/보행자 라우팅 API로 교체 가능한 구조로 설계.
- **GPS 시뮬레이터:** 실기기 없이 데모하도록 경로를 따라 걷는/배회하는 시뮬 제공.
- **지도 타일:** TMAP JS SDK로 렌더(밝은 타일). 서버 보행자 경로안내 키는 미구독 → 실제 도로 스냅 없음.
- **인증/백엔드 없음:** 소셜 로그인은 UI만, 데이터는 브라우저 `localStorage`에 로컬 저장.
- **미구현:** 여기 문제 핀 신고, 코스 명명/편집, 좋은 날 푸시, 기여 임팩트, 어드민, 개인화 학습.
