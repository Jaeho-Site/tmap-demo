## Phase 1 — 지도 단일 경로 선택 (F1)  ·  규모 S

**현상태:** `TmapMap`은 모든 코스 폴리라인을 목적별 색으로 그리고, `selectedId`가 있으면 그 위에 라임색 하이라이트를 **덧그린다**(다른 초록선은 그대로 남음). 목적 필터만 폴리라인 표시/숨김을 제어한다.

**목표:** 홈의 "주변경로탐색"처럼 하나를 고르면 **그 경로만**, 취소하면 **전체 복원**.

- [ ] `TmapMap` 표시 로직 수정: `selectedId`가 있으면 선택된 코스 외 **베이스 폴리라인 전부 숨김**(`setMap(null)`), 하이라이트만 노출
- [ ] 선택 해제(`selectedId = null`) 시 **기존 목적 필터 규칙대로** 전체/필터 폴리라인 복원
- [ ] 필터 이펙트와 선택 이펙트의 표시 우선순위 정리(선택 > 필터). `filterPurpose`·`selectedId` 두 이펙트가 서로 덮어쓰지 않도록 단일 "가시성 계산" 함수로 통합
- [ ] `MapExplore`의 하단 주변코스 패널 선택/해제와 지도 클릭 선택이 동일 상태(`selectedId`)를 공유하는지 확인(이미 `select` 토글 공유 중)
- **DoD:** 탐험>지도에서 초록선 하나(또는 하단 패널 항목)를 누르면 지도에 그 경로만 남고, 다시 누르면(선택 취소) 전체 초록선이 복원된다. 목적 필터와 조합해도 깨지지 않는다.
- **참고:** 모험 탭의 AI 추천/저장 경로 단일 표시는 이미 `guidePath`로 동작 → 이 방식과 일관되게 맞춘다.

## Phase 2 — 산책 파트너(강아지) 가챠 시스템 (F2)  ·  규모 L

> 보상 → 상자 → 강아지 뽑기(해상도=희귀도 D·C·B·A·S) → 확정 상자, 그리고 **파트너 관리 탭**.
> 규모가 커서 4개 서브페이즈로 분할. 2.1(엔진)→2.2(뽑기 연출)→2.3(관리 탭)→2.4(확정/천장) 순.

### 데이터 모델(초안)
```ts
type Rarity = 'D' | 'C' | 'B' | 'A' | 'S';       // 낮음→높음
interface DogBreed { id; name; rarity: Rarity; image; desc; }   // 도감 마스터
interface OwnedDog { uid; breedId; rarity; obtainedAt; nickname?; } // 보유 인스턴스
type BoxKind = 'normal' | 'guaranteed';           // 일반 / 확정(예: A 이상 확정)
interface PartnerState {
  boxes: { normal: number; guaranteed: number };  // 미개봉 상자 수
  dogs: OwnedDog[];                                // 보유 강아지
  activePartnerUid?: string;                       // 대표 파트너
  pity: number;                                    // 천장 카운터(누적 뽑기)
}
```
가챠 확률(예시, 조정 대상): D 45 / C 30 / B 17 / A 6 / S 2 (%). 확정 상자는 최소 A 보장.

### Phase 2.1 — 파트너 엔진 & 보상 획득  ·  규모 M
- [ ] `features/partner/breeds.ts`: 강아지 마스터 데이터(희귀도별 견종·이미지 플레이스홀더)
- [ ] `features/partner/gacha.ts`: 가중 랜덤 뽑기, 확정/천장 규칙, 순수 함수(테스트 용이)
- [ ] `store/partner.ts`(zustand + localStorage): 상자·보유견·대표 파트너·pity 영속화
- [ ] **보상 획득 훅**: 산책 완주 시 상자 지급(예: 완주 1회=일반 상자 1, N일 연속/거리 마일스톤=확정 상자). `WalkCompletePage`/`records`에 연결
- **DoD:** 완주하면 상자가 쌓이고, 뽑기 함수가 확률대로 강아지를 반환하며 상태가 localStorage에 유지된다(단위 확인).

### Phase 2.2 — 상자 개봉 · 뽑기 연출  ·  규모 M
- [ ] `pages/PartnerPage`(또는 시트): 미개봉 상자 표시 → "열기" → **개봉 연출**(상자 흔들림 → 광채 → 강아지 공개)
- [ ] 희귀도별 **연출 차등**(D~C 담백 / B~A 강조 / S 특별 광채·색). `DESIGN_SYSTEM` 라임 액센트 유지
- [ ] 중복 획득 처리(중복 시 재화/조각 지급 등 규칙 — MVP는 단순 보유 누적도 가능)
- [ ] 완주 직후 진입 동선: `WalkCompletePage`에 "상자 열러 가기" CTA
- **DoD:** 상자 열기 → 희귀도에 맞는 연출과 함께 강아지가 공개되고 보유 목록에 추가된다.

### Phase 2.3 — 파트너 관리 탭  ·  규모 M
- [ ] 진입점 결정(택1, 아래 "확인 필요"): (a) 하단 6번째 탭 신설 / (b) 탐험 토글에 `파트너` 추가 `[지도|도감|모험|파트너]` / (c) 프로필 하위 페이지 `/partners`
- [ ] 강아지 **도감/컬렉션 뷰**: 보유/미보유(실루엣), 희귀도 뱃지(D·C·B·A·S), 획득일
- [ ] **대표 파트너 지정** + 홈/프로필/산책 화면에 대표 강아지 노출(리텐션 훅)
- [ ] 닉네임 등 간단 관리(선택)
- **DoD:** 파트너 탭에서 보유 강아지를 희귀도별로 보고, 대표 파트너를 지정하면 다른 화면에 반영된다.

### Phase 2.4 — 확정 상자 & 천장(pity)  ·  규모 S
- [ ] 확정 상자 = 특정 등급 이상 보장(예: A 이상). 획득 조건 정의(마일스톤/연속기록)
- [ ] 천장: 누적 N회 미획득 시 S 확정 등 pity 로직을 `gacha.ts`에 반영
- [ ] 확률/천장 **정직한 표기**(UX P7 톤) — 상자 상세에 확률 공개
- **DoD:** 확정 상자는 항상 보장 등급 이상을 주고, 천장 도달 시 상위 등급이 확정된다. 확률이 화면에 공개된다.

## Phase 3 — 개인 산책로 공유 & 검토 프로세스 (F3)  ·  규모 M

> 개인 소장이던 커스텀 경로를 **공유**로 전환 → **관리자 검토**(검토중 → 승인/반려) 상태 표시.
> 백엔드가 없으므로 **검토는 목업**(개발용 승인 버튼 또는 시간 경과 자동 승인)으로 먼저 만들고 인터페이스만 서버용으로 고정한다.

### 데이터 모델(customRoutes 확장)
```ts
type ShareStatus = 'private' | 'pending' | 'approved' | 'rejected'; // 미공유·검토중·승인·반려
interface CustomRoute {
  /* 기존 필드 + */
  shareStatus: ShareStatus;   // 기본 'private'
  sharedAt?: number;
  usageCount?: number;        // Phase 4에서 사용
  authorName?: string;        // 공유 목록 표기용(프로토타입: 로그인 provider)
}
```

- [ ] `customRoutes.ts`에 `shareStatus` 필드 + `share(id)`/`unshare(id)` 액션 추가(마이그레이션: 기존 항목 `private` 기본값)
- [ ] `SavedPage` "내가 만든 경로" 카드에 **공유하기** 액션 + **상태 뱃지**(검토중/승인/반려) 노출
- [ ] 상태별 UI: 검토중(회색·시계), 승인(라임·체크), 반려(사유 표시). 승인 전엔 공유 목록 미노출
- [ ] **검토 목업 처리**(택1): (a) 개발자용 "승인 시뮬" 토글 / (b) 공유 후 일정 시간·새로고침 시 자동 승인. 실제 API 자리(`reviewRoute`)는 인터페이스만 정의
- [ ] (선택) 공유 시 최소 검증 안내(개인정보·위치 노출 유의 카피)
- **DoD:** 커스텀 경로를 공유하면 "검토중"으로 표시되고, (목업)승인 후 "승인"으로 바뀌며 공유 대상이 된다. 상태가 localStorage에 유지된다.
- **⚠️ 실서비스 필요:** 공유 경로 저장/검토 큐/관리자 콘솔(백엔드). 프로토타입은 단일 사용자 목업.

## Phase 4 — 공유 산책로 인기 랭킹 (F4)  ·  규모 S~M

> 승인되어 공유된 산책로 중 **사용수(사용 횟수) 상위** 순위 노출. Phase 3 데이터 위에 구축.

- [ ] `usageCount` 증가 규칙: 공유 경로를 불러와 산책 시작/완주 시 +1(프로토타입은 본인 사용 기준). `AdventureView`의 `?route=` 진입/완주 지점에 연결
- [ ] **시드 데이터**: 단일 사용자 프로토타입이라 랭킹이 비지 않도록 인기 공유 경로 목업 N개(사용수 포함) 제공 + 사용자의 승인 경로 병합
- [ ] **랭킹 뷰**: 사용수 내림차순 정렬. 위치는 택1 — 홈 섹션 "인기 공유 산책로" / 탐험 토글 신규 섹션 / 별도 `/ranking`
- [ ] 랭킹 카드: 순위·경로명·거리/시간·사용수·작성자, 탭 시 지도에서 해당 경로 보기(모험 `?route=` 재사용)
- **DoD:** 승인·공유된 경로가 사용수 내림차순 랭킹으로 표시되고, 사용할수록 순위가 갱신된다.
- **⚠️ 실서비스 필요:** 다중 사용자 사용수 집계(서버). 프로토타입은 목업+본인 카운트.

## 5. 의존성 & 마일스톤

```
F1(지도) ────────────────► 즉시 릴리스 가능(독립, 규모 S)
F3(공유·검토) ──► F4(랭킹)   ← F4는 F3의 공유/승인 데이터에 의존
F2(파트너 가챠) ─────────────► 독립 대형 에픽(2.1→2.2→2.3→2.4). 다른 트랙과 병렬 가능
```

| 마일스톤 | 포함 | 사용자 체감 |
|---|---|---|
| M1 (빠른 개선) | F1 | 지도에서 경로 하나만 깔끔히 보기 |
| M2 (커뮤니티 씨앗) | F3 → F4 | 내 경로 공유·검토 → 인기 경로 랭킹 |
| M3 (리텐션 게임화) | F2 (2.1~2.4) | 산책 보상으로 강아지 수집·파트너 |

## 6. 결정/확인이 필요한 사항 (사용자에게)

- **F2 파트너 탭 위치:** ⓐ 하단 6번째 탭 신설 / ⓑ 탐험 토글에 `파트너` 추가 / ⓒ 프로필 하위 `/partners` — 어디로?
- **F2 상자 지급 규칙:** 완주 1회=일반 상자 1개가 기본. 확정 상자 획득 조건(연속기록? 거리 마일스톤? 주간 목표?)은?
- **F2 가챠 확률/천장 수치:** 예시값(D45/C30/B17/A6/S2, 천장 N회) 사용 여부 및 조정.
- **F2 강아지 이미지 에셋:** 희귀도별 견종 이미지가 있는지(없으면 플레이스홀더/AI 생성 그림으로 진행).
- **F3 검토 목업 방식:** 개발자 승인 토글 vs 시간 경과 자동 승인.
- **F4 랭킹 노출 위치:** 홈 섹션 / 탐험 신규 섹션 / 별도 페이지.

> 위 답이 없어도 각 Phase는 합리적 기본값(위 예시)으로 진행 가능하며, 확정 시 값만 교체한다.
