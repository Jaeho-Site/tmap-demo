# 산책온 · 대전 MVP 지도 — 작업 보고서

> 작성일: 2026-07-06 · 프로젝트: `map-test/` (React 19 + TypeScript + Vite)
> TMAP 지도 위에 대전 산책 데이터를 노드/경로로 얹은 시제품

---

## 0. 한 줄 요약

대전 중심 TMAP 지도에 **보고서 4대 산책 목적 데이터 + 도시공원 465개 + 가로수길 314개**를 노드로 올리고,
가로수길은 **TMAP 보행자 경로 API로 시작→종료를 실제 보행 경로로 연결**했다.

## 2. 데이터 소스와 처리

### 2-1. 보고서 기반 지점 (4대 목적)
`src/data/daejeonData.ts` — API 검증 보고서에서 추출한 실데이터를 마커로.

| 목적 | 데이터 | 지도 표현 |
|---|---|---|
| 🛡️ 방범·안전 | 가로등 43,082건·CCTV·보행자 사고통계(52%/55%) | 사이드바 통계·데이터셋 카드 |
| 🌳 자연친화 | 대전둘레산길 12구간 대표 구간 | 핀 마커 |
| 🦵 관절보호 | TMAP 육교 실증(계룡로, turnType 125) | 핀 마커 |
| 🤫 조용함 | 국가소음 6지점(야간 dB) | dB별 색상 핀 |

> 좌표 없던 항목(소음·둘레산길)은 주소/장소 기반 **근사 좌표**로 표시하고 팝업에 명시.

### 2-2. 도시공원 노드 — 465개
- 소스: `data/대전광역시_*_도시공원정보_*.csv` (5개 구)
- 생성: `scripts/generateParks.mjs` → `src/data/parks.ts`
- **위경도 유효 465개 전부**를 원형(●) 노드로. 좌표 무효 제외 0건.
- 태그: 공원구분 + 시설(운동/놀이/편의/교양/화장실) 자동 부여.
- 구분별 분포: 어린이공원 277 · 근린공원 84 · 소공원 64 · 문화공원 16 · 체육공원 10 · 수변공원 7 · 기타 5 · 역사공원 2
- 사이드바에서 **구분별 칩 필터** 지원.

### 2-3. 가로수길 노드 + 경로 — 314개
- 소스: `data/가로수길/대전광역시_*_가로수길정보_*.csv` (4개 구)
- 생성: `scripts/generateStreets.mjs` → `src/data/streets.ts`
- **시작·종료를 각각 노드로**: 시작(◆ 마름모) / 종료(● 원), 수종별 색상.
- 좌표 무효 5건 제외 — 원본 CSV에 **서울권 좌표(37.x)가 잘못 입력**된 행(서구 도산·만년·문예·문정로, 중구 충무로).

---

## 3. 핵심: TMAP 보행자 경로 연결

- **재검증 결과, 제공된 TMAP 키가 REST 보행자 경로안내 API에서 정상 동작**
  (초기 보고서의 `INVALID_API_KEY`는 현재 해소됨).
- `scripts/fetchStreetRoutes.mjs` 가 314개 구간의 시작→종료를
  `POST /tmap/routes/pedestrian` 로 호출 → 실제 보행 경로(LineString) 좌표를
  `src/data/streetRoutes.ts` 로 **빌드 시 미리 구움**(런타임 314회 호출 회피).
- 결과: **291/314 (93%)** 실제 보행 경로(실선), 나머지 23건은
  TMAP 라우팅 불가(에러 1107, 외곽·산지 보행망 없음)로 **직선 점선 폴백**.
- 노드 팝업에 보행 거리·소요시간 표시 (예: 🚶 보행 819m · 약 13분).

## 4. 화면 구성

- **좌측 사이드바**: 4대 목적 레이어 토글 · 도시공원(구분 칩) · 가로수길 · 안전통계 · 데이터셋 출처
- **지도**: 목적 핀 / 공원 원형 / 가로수길 시작◆·종료● + 보행경로 폴리라인
- 모든 노드 클릭 → 상세 팝업, 사이드바 항목 클릭 → 해당 좌표로 이동

## 5. 산출물

```
map-test/
├─ data/                         # 원본 CSV (공원 5 + 가로수길 4)
├─ scripts/
│  ├─ generateParks.mjs          # 공원 CSV → parks.ts
│  ├─ generateStreets.mjs        # 가로수길 CSV → streets.ts
│  └─ fetchStreetRoutes.mjs      # TMAP 보행경로 precompute → streetRoutes.ts
├─ src/
│  ├─ data/  daejeonData / parks / parkMeta / streets / streetMeta / streetRoutes
│  ├─ components/  MapView.tsx · Sidebar.tsx
│  ├─ hooks/  useTmapSdk.ts
│  └─ config.ts · tmap.d.ts
└─ index.html                    # TMAP JS SDK 동기 로드
```

| 노드 종류 | 개수 |
|---|---:|
| 보고서 지점(소음·둘레산길·육교 등) | 12 |
| 도시공원 | 465 |
| 가로수길(시작+종료) | 314 × 2 = 628 |
| 가로수길 보행경로 | 291 (직선 폴백 23) |

## 6. 남은 개선 여지

- 경로 좌표가 많아 번들 gzip 226KB → 경로 단순화(Douglas–Peucker) 또는 lazy-load
- 줌 아웃 시 노드 겹침 → 마커 클러스터링
- 공원·가로수길 노드를 잇는 **코스(엣지) 그래프**로 산책길 추천 확장
- 실패 23개 구간 및 서울권 오좌표 5건은 원본 데이터 정정 필요
